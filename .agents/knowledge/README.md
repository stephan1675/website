# LLM Knowledge Base

Welcome to your LLM Knowledge Base. This directory holds project context, architectural decisions, and other training data/documentation to help AI agents understand the codebase instantly.

## Recommended Structure

To keep information organized, structure this directory as follows:

1. **`architecture/`**: High-level designs, database schemas, component boundaries, and API definitions.
2. **`guidelines/`**: Specific programming style guides, patterns to replicate, or patterns to avoid.
3. **`decisions/`**: Architecture Decision Records (ADRs) explaining *why* certain technologies or structures were chosen.
4. **`context/`**: Business domain context, project goals, glossary of terms, and reference materials.

## Guidelines for Adding Files

- Keep files in standard Markdown (`.md`) format.
- Keep documents concise, using lists and headers.
- Update these documents whenever a major structural or architectural change is made.
